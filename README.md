# AutoPM - Minimal Process Data Agent

AutoPM — это легковесный агент для быстрой первичной загрузки и оценки структуры данных бизнес-процессов.

## 🚀 Возможности
*   **Загрузка данных**: Поддержка CSV и Excel файлов.
*   **Автоматическое форматирование**: Агент (через LLM) анализирует данные и автоматически конвертирует числа и даты в правильный формат.
*   **Базовый отчет**: Генерирует простой отчет (`reports/report.md`) с количеством строк, столбцами и их названиями.
*   **Приватный Чат**: Интерфейс для вопросов о **структуре** данных. Агент знает только метаданные (названия колонок) и намеренно отказывается от выполнения кода или глубокой аналитики, обеспечивая безопасность и простоту.

## 🚦 Быстрый старт

### 1. Установка
```bash
git clone <repository_url>
cd AutoPM
pip install -r requirements.txt
```

### 2. Конфигурация
В корне проекта находится файл `config.yaml`. Отредактируйте его, указав ваши настройки:
```yaml
# Пример config.yaml
provider: "mistral" # или "local"

mistral:
  api_key: "ВАШ_API_КЛЮЧ" 
  model: "mistral-medium-latest"

local:
  base_url: "http://localhost:11434/v1"
  model: "llama3.2"
  api_key: "ollama"
```

### 3. Запуск
```bash
python pm_agent/main.py --file "data.csv"
```
Или просто запустите `python pm_agent/main.py` и следуйте инструкциям.

После обработки данных вы попадете в чат, где сможете спросить:
*   *"Какие есть колонки?"*
*   *"Сколько всего записей?"*

## 📊 Логика анализа отклонений

Агент автоматически находит 18 типов отклонений. Ниже описано, как они рассчитываются:

### 1-6. Циклы и Аномалии
- **1. Аномалии (Outliers)**: Метод IQR для длительности кейсов. Кейсы > `Q3 + 1.5 * IQR`.
- **2-3. Self-loops**: Поиск одинаковых событий подряд: `df[act] == df[next_act]`.
- **4. Returns**: Повтор события через другие шаги (A -> B -> A).
- **5. Ping-pong**: Чередование двух этапов (A -> B -> A).
- **6. Back-to-start**: Возврат из середины процесса на первый шаг.

### 7-12. Временные показатели
- **7. Long Cycles**: Кейсы, попадающие в топ-5% по общей длительности.
- **8. Bottlenecks**: Переходы с наибольшим средним временем ожидания.
- **9. Manual Steps**: Этапы с коэффициентом вариации `(std/mean) > 0.5`.
- **10-11. Инциденты**: Операции в 2 раза дольше среднего (`duration > 2 * mean`).
- **12. Deadlines**: Превышение заданного порога или попадание в 5% самых долгих.

### 13-18. Качество и Скрытые процессы
- **13. Critical Steps**: Корреляция Пирсона между длительностью этапа и всего кейса `> 0.5`.
- **14. Redundant Activities**: Т-тест: если наличие шага не меняет общую длительность (p > 0.05).
- **15. Variability**: Отношение количества вариантов к общему числу кейсов `> 0.3`.
- **16. Dark Processes**: Варианты, не входящие в 80% самых частых сценариев.
- **17. Manual Exceptions**: Длительные этапы с низкой вариативностью (`CV < 0.3`).
- **18. Rework Loops**: Переходы "назад" относительно среднего порядка этапов.
